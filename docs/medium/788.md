# 788. Rotated Digits

## Question

An integer `x` is a **good** if after rotating each digit individually by 180 degrees, we get a valid number that is different from `x`. Each digit must be rotated - we cannot choose to leave it alone.

A number is valid if each digit remains a digit after rotation. For example:

- `0`, `1`, and `8` rotate to themselves,
- `2` and `5` rotate to each other (in this case they are rotated in a different direction, in other words, `2` or `5` gets mirrored),
- `6` and `9` rotate to each other, and
- the rest of the numbers do not rotate to any other number and become invalid.

Given an integer `n`, return *the number of good integers in the range* `[1, n]`.

### Example 1:
```
Input: n = 10
Output: 4
Explanation: There are four good numbers in the range [1, 10] : 2, 5, 6, 9.
Note that 1 and 10 are not good numbers, since they remain unchanged after rotating.
```

### Example 2:
```
Input: n = 1
Output: 0
```

### Example 3:
```
Input: n = 2
Output: 1
```

### Constraints:
- `1 <= n <= 104`

## Solution
```java
class Solution {
    public int rotatedDigits(int n) {
        int good = 0;

        for (int i = 1; i <= n; i++) {
            int num = i;
            int rotateNum = 0;
            int count = 0;
            boolean invalid = false;

            while (num > 0) {
                int current = num % 10;
                if (current == 3 || current == 4 || current == 7) {
                    invalid = true;
                    break;
                }

                switch (current) {
                    case 2:
                        current = 5;
                        break;
                    case 5:
                        current = 2;
                        break;
                    case 6:
                        current = 9;
                        break;
                    case 9:
                        current = 6;
                        break;
                }

                rotateNum += current * Math.pow(10, count);
                count++;

                num = (int) (num / 10);
            }

            if (invalid) {
                continue;
            }

            if (rotateNum != i) {
                good++;
            }
        }

        return good;
    }
}
```

## Mindset
The question mentions that a `good number` should fulfill the following conditions:

1. The number remains valid after rotating each digit individually by 180 degrees.
2. The rotated number is different from the original number.

This means a `valid number` may not be a `good number`.

Here are some `valid and good number` examples:

1. 1265 rotates to 1592
    - 1 → 1
    - 2 → 5
    - 6 → 9
    - 5 → 2
2. 9255 rotates to 6522
    - 9 → 6
    - 2 → 5
    - 5 → 2

The following numbers are `valid numbers`, but they are not `good numbers` because they remain the same after rotation:

1. 18 rotates to 18
2. 1880 rotates to 1880

Here are some invalid number examples (i.e., the number contains 3, 4, or 7):

1. 17
2. 2347

So, this question requires splitting and converting every digit one by one. It uses the modulo operator to extract the rightmost digit from the original number. For example, `2591 % 10 = 1`. Then, it checks whether the rightmost digit is invalid (i.e., 3, 4, or 7). If it is, the number is invalid and can be skipped. Otherwise, it rotates the digit and adds it to `rotateNum`. Since the splitting process starts from the rightmost digit, directly appending digits to `rotateNum` would reverse the number. Therefore, it uses `Math.pow` to place each rotated digit in the correct position. Finally, it divides the original number by 10 to get the next rightmost digit. This process repeats until the original number is 0. Then, it checks whether the rotated number is equal to the original number. If it is not equal, the number is a good number.
